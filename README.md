# ルール
## リポジトリ命名規則

基本フォーマット：{system}-{component}-{kind}
※リポジトリ命で何が管理されているかがわかるのが理想
※Web/APIなど両方が入っている場合等はメイン（主観）に寄せる

| 要素 | 内容 | 例 |
| --------- | ------- | --------------------------- |
| system | システム名 | cableidpf/ml/mvno-uq |
| component | 機能・構成要素 | api/web/common/native-app |
| kind | 管理対象    | source/docs |
